Jquery checkform v1.2

Jquery checkform plugin !

Tired of javascript form validation ? This Jquery plug-in provides various HTML form validation styles (events, ui styles, ajax...). It includes also a set of default patterns for the most common fields to be checked.

See examples

Getting started

Add the following lines at the end of the HTML body
<script type="text/javascript" src="jquery.min.js"></script> <script type="text/javascript" src="jquery.checkform.js"></script> For the following HTML form : <form action="#" id="registry1" method="post"> <fieldset> <label for="email">Mail address</label>: <input name="email" id="email" type="text" /> <label for="password">Password</label>: <input name="password" id="password" type="text" /> <input id="register1" name="r1" value="Register" type="submit"/> </fieldset> </form> Initialize the checkform plugin : $('#registry1').checkform(); Your form will now be checked before being submitted. For more details about how default fields are checked see the default pattern section.

Configuration options

List of options
Name Possibilities Default value Effect Plug-in version
autoCheck true, false true Determines if the plug-in has to automaticaly check the default items. If set to false, inputs must be specified in the items option 1.0+
CSSClass A correct class name checkform-wrong Specifies the class which will be apply to the interface effect (parameter "type"). Several classes can be added by passing a string containing space-separated classes. 1.0+
afterValidate function null A function to be executed after the form has been validated. If at least one field is wrong, this function won't be executed. 1.1+
event change, lostfocus, submit submit Determines when fields have to be checked.
  • change : a field is checked when its value changes (a key is pressed).
  • lostfocus : a field is checked when the focus leave it.
  • submit : all field are validated when the form is submitted.
Note : Submit checking is always called. If at least one field is wrong, data won't be end.
Warning : Be careful using both event change and ajax checking for performance reasons.
1.0+
language en, fr, ge, sp en Specifies the language of the message displayed.
  • "en" for englsih
  • "fr" for french
  • "ge" for german
  • "sp" for spanish
Note : A new language can be added at the top of javascript file by adding a key to the checkform_message array
1.0+
type flash, input, label, text text Determines which kind of user inteferface has to be applied when a field is wrong. The CSSClass option is added to the UI element.
  • text : A message is displayed at the end of the form containing all wrong item's label.
  • flash : Same as text but add a close message button.
  • label : add the css class to the previous label of the wrong item
  • input : add the css class to the wrong item
1.0+
items Allows to specify new kind of items or to overwrite a default item. You can add a MAC adress pattern or a credit card one.
An item is defined by four properties :
  • selector : a CSS selector to determine which fields in the have to be checked
  • pattern : a regular expression to the field
  • autoTrim : if true, field will be trimed before being checked.
  • ajaxCheck : TO BE DONE
See the default values here
Example : 
"items":{
  "MACAdress":{ //Defines a new kind of check
      "selector":"input.macAdress",
      "pattern':'^([0-9A-F]{2}:){6}$'
  },
  "birthday":{ //Specifies a new rule for bithday items
      "pattern":"^[0-9]{2}-[0-9]{2}-[0-9]{4}$"
  }
}
Warning : For a new item, selector and pattern are require. autoTrim and ajaxCheck options will be respectively set to true and false by default.
1.0+
