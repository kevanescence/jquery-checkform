Why using checkform plug-in ?

This plug-in aims you at developping faster and making your form validation less tedious. It enables you to make various user interfaces in javascript easily. Moreover it prevents server from being too busy by checking forms. Indeed, by using this plug-in, server will check only one time data.

How to use ?

How it works

In order to avoid compatibilty issues, please use the latest jQuery version.

The checkform plug-in uses name html attribute in each input tags. In this way if you respect conventional names the initialization will be done automaticaly. For example an input called "name" will be check with the corresponding regular expression. In this way you don't have to be an expert in RegEx to use the plug-in

You will be able to chose different kind of UI action that have to be display when some fields are wrong. There are 4 types of effect : label, input, flash or text (default setting).

Different event are also provided. In this way data can be checked as soon as a letter is type or when input loses focus or more tradionaly when the form is sent.

Arguments must be transmitted as a JSON expression
Important : all field must have a label even if it is not displayed. It enables the plug-in to display an explicit message to indicate wrong fields.

File dependencies

Please add the following lines in the header tag of your file <script type="text/javascript" src="jquery.min.js"></script> <script type="text/javascript" src="jquery.checkform.js"></script>

List of options available

Name List of possibility Default value Effect Plug-in version
autoCheck True, False True Set if the plug-in has to automaticaly check the name html attribute of input tags. If set to false, inputs must be specified in items option 1.0+
CSSClass A correct class name checkform-wrong Specifies the class which will be apply to the interface effect (parameter "type"). 1.0+
afterValidate function null A function to be executed when form is validated (submit event) 1.1+
event change, lostfocus, submit submit Determine when fields have to be checked.
If set to submit, fields will only be verified before sending data. If at least one incorrect field, data won't be sent.
If set to lostfocus, input text will be checked as soon as input loses focus.
If set to change, the field will be checked at every time it changes. Note that submit checking is called in every case. Please notice too that change event requires more performance than submit one. 		1.0+
language en, fr, ge, sp en Specifie the language of the message displayed. "en" for englsih, "fr" for french, "ge" for german and "sp" for spanish. A new language can be added at the top of javascript file. Add a key to the checkform_message array 1.0+
type flash, inputs, label, text text Determine which kind of user inteferface you want to use. The effect will have CSSClass parameters for css class. In this way css lines allow more personalization.
Value "flash" add a div in the html file. The div will have <CSSClass> class.
Value "text" will add a span tag at the end of the form and indicate the label of wrong fields.
Value "input" will simply add <CSSClass> class to the wrong inputs.
Value "label" will add <CSSClass> to the wrong field's previous label. 		1.0+
items You can specify here items to be checked if default selectors don't satisfy you.
You can also use this part in order to specify new pattern for a group of input. Indeed you might want to define a rule which is not in the default settings. For example you can add a MAC adress pattern or a credit card one.
Item is defined by two property : a selector and a pattern. Each one of these settings has a default value so you can overwrite only one property or both. Specify this options like this : 
"items":
                                {
                                "MACAdress":{ //Define a new kind of check
                                              "selector":"input.macAdress",
                                              "pattern':'^([0-9A-F]{2}:){6}$'
                                              },
                                "birthday":{ //Specify a new rule for bithday
                                            "pattern":"^[0-9]{2}-[0-9]{2}-[0-9]{4}$"
                                            }
                                }
1.0+
